  • Developed a same-day automated remediation for the 2024 CrowdStrike global outage and deployed it across multiple enterprise clients with a greater than 93% success rate. The core challenge was the boot loop — which cut off most endpoint management solutions entirely. The solution leveraged MECM's PXE boot capability to bring affected devices into a stable WinPE environment, then executed a fully automated remediation sequence. Two constraint scenarios required separate handling: BitLocker-encrypted drives could not be accessed from WinPE without first unlocking them. To solve this, an isolated copy of the BitLocker recovery key database was created and a conditional unlock script was written to query it via a temporary SQL account — allowing each device to retrieve its own key without exposing or writing new keys to the live environment. A separate key reroll script was then registered as a scheduled task on the offline OS, forcing a new recovery key to be generated and escrowed on first boot back into Windows — closing the exposure window cleanly. The full task sequence ran: BitLocker unlock (if applicable) → CrowdStrike file remediation via PowerShell matching CrowdStrike's official resolution pattern → scheduled task registration for key reroll on reboot. The only manual step was the PXE boot entry point — reboot, press F12, select network boot — simple enough to be performed by any employee, not just IT staff. Because the solution contained no sensitive data or hardcoded secrets it was portable and deployable across any number of clients or organizations
